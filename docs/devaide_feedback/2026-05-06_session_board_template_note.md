# DevAide Note: Session Board Template Candidate

作成日: 2026-05-06
作成者: Codex
対象プロジェクト: 投資家タイプ診断
ステータス: DevAide本体未反映 / テンプレート候補

## 目的

ChatGPTから提案された `DevAide Session Board` を、DevAide本体へ反映しやすい形に整理する。

この文書は、マルチAI開発でユーザーが現在地を見失わないための「見える状態管理」テンプレート候補である。

## 元案の評価

元案は非常に有用。

理由:

- ユーザーが一瞬で開発状況を把握できる。
- AI担当ごとの責任範囲が見える。
- ブロッカーと次アクションが分かれる。
- DevAideが防いだ混乱を自然に可視化できる。
- DevAideが前に出すぎず、セッション保護の価値だけを伝えられる。

これは、DevAideの価値である「起きなかった事故」をユーザーに伝えるための軽量なUI/ドキュメントとして使える。

## 改善方針

元案をDevAideテンプレートにするなら、以下を整えると実運用に向く。

- 状態表記を `完了 / 要確認 / 未着手 / 進行中` で統一する。
- 残り作業はチェック項目化する。
- AI担当には、名前だけでなく責任範囲を書く。
- 次にやることには担当者を入れる。
- `DevAideが防いだ混乱` は必ず残す。
- ただし、DevAideの成果を過剰に主張しない。

## テンプレート案

```md
# DevAide Session Board

## 目的
このセッションで達成したい状態を書く。

## 現在の状態
- 実装: 未着手 / 進行中 / 要確認 / 完了
- 画像: 未着手 / 進行中 / 要確認 / 完了
- ロジック: 未着手 / 進行中 / 要確認 / 完了
- コンテンツ: 未着手 / 進行中 / 要確認 / 完了
- QA: 未着手 / 進行中 / 要確認 / 完了
- 公開: 未着手 / 進行中 / 要確認 / 完了

## AI担当
- Claude Code: 実装 / 修正
- Codex: レビュー / 品質確認 / 記録
- ChatGPT: 方針整理 / UX判断
- User: 最終判断 / 公開判断

## ブロッカー
- なし

## 次にやること
1. 担当者: 次の具体タスク
2. 担当者: 次の具体タスク
3. 担当者: 次の具体タスク

## DevAideが防いだ混乱
- 最新成果物の迷子化
- working成果物とactive成果物の混同
- 実装担当とレビュー担当の役割混線
- 公開前チェック項目の抜け漏れ
```

## DevAide本体への反映案

追加候補:

```text
docs/SESSION_BOARD_TEMPLATE.md
```

関連付けたい既存/候補ドキュメント:

```text
docs/SESSION_HEALTH_SUMMARY_TEMPLATE.md
docs/PREVENTED_RISK_LOG_TEMPLATE.md
docs/SESSION_HANDOFF_TEMPLATE.md
docs/AI_START_ROUTER_TEMPLATE.md
```

## 使うタイミング

以下のタイミングで更新すると有効。

- 開発開始時
- 担当AIが増えた時
- 作業場所やactive成果物が決まった時
- 公開前チェックに入った時
- ユーザーが「次は？」と聞いた時
- セッション引き継ぎ前

## 注意点

- 長くしすぎない。
- 毎ターン更新しない。
- ユーザーに見せる前提で簡潔にする。
- DevAideの成果アピールではなく、セッションの安全状態を伝える。
- `DevAideが防いだ混乱` は根拠のあるものだけを書く。

## 結論

`DevAide Session Board` は、今回の開発で特に有用そうなテンプレート候補。

DevAideの「見えない保護」をユーザーに自然に伝えるため、`Session Health Summary` と並んで優先度高で反映する価値がある。
