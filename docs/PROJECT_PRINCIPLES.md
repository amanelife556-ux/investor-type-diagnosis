# Project Principles: 投資家タイプ診断

最終更新日: 2026-05-05
管理者: ユーザー

## このツールは何のためのものか

20問・4軸から、ユーザーの投資スタイルを16タイプの「投資家タイプ」として自己理解しやすく見せる診断アプリです。

目的:

- 投資スタイルの傾向を楽しく把握できるようにする。
- 結果画面で、強み、気をつけたいクセ、振り返りポイントを見せる。
- SNS共有しやすい短い診断結果を生成する。

目指さないこと:

- 個別銘柄や金融商品の推奨。
- 投資助言、売買判断、利益保証。
- 口座連携、個人情報保存、複雑な分析ツール化。

## 優先する価値

1. 投資助言に見えない安全な自己理解表現
2. 16タイプ診断としての納得感
3. スマホで最後まで進めやすい導線
4. ビジュアルの世界観と結果画面の満足感
5. 変更しやすい責務分離

迷った場合は、機能追加よりも「診断体験が壊れないこと」と「投資助言に見えないこと」を優先してください。

## AIはどこまで判断してよいか

AIが判断してよいこと:

- 既存仕様に沿った小さなUI/CSS修正。
- テスト追加。
- READMEや引き継ぎメモの更新。
- 表示崩れやビルド失敗の最小修正。

AIが人間に確認すべきこと:

- 診断軸の意味変更。
- 設問数や16タイプ構造の変更。
- タイプコードや判定ロジックの変更。
- 結果画面の導線や共有仕様の大きな変更。
- 依存パッケージの追加。

AIが判断してはいけないこと:

- 投資助言に近い機能の追加。
- 対象ユーザーやプロダクト方向性の変更。
- テストや禁止事項の削除・弱体化。

## AI役割分担

| 役割 | 担当 | 補足 |
|---|---|---|
| 全体構成/統合レビュー | Codex | 構成、責務分離、最終判断の整理 |
| ビジュアル担当 | 未固定 | `src/styles.css` と `assets/` |
| ロジック担当 | 未固定 | `src/diagnosisLogic.ts` と `src/__tests__/` |
| コンテンツ担当 | Fermat | 設問、結果文、安全表現 |
| QA担当 | Raman | テスト、ビルド、手動確認 |

## 設計判断モードに入る条件

以下に当てはまる場合は、実装を止めて人間に確認してください。

- 16タイプ以外に増やす/減らす。
- 4軸の意味を変える。
- `value: -1 | 1` の意味を変える。
- 結果を投資助言や金融商品推奨に近づける。
- 外部サービス連携やデータ保存を追加する。
- 画像生成や再書き出しでタイプ表現の方向性を変える。

## 壊してはいけない前提

- `src/main.tsx` は起動専用。
- 画面本体は `src/App.tsx`。
- 診断データは `src/diagnosisData.ts`。
- 診断ロジックは `src/diagnosisLogic.ts`。
- 16タイプコードと `assets/type-portraits/{TYPE_CODE}.png` は対応している。
- `npm test` と `npm run build` が公開前の必須確認。

## 過剰実装してはいけない部分

- ログイン、保存、分析ダッシュボード。
- 個別投資商品の提案。
- 複雑な状態管理ライブラリ導入。
- 診断結果の過度なパーソナライズ。

## 禁止事項

- 投資成果を保証する表現を追加しない。
- 「買うべき」「売るべき」「おすすめ銘柄」などの助言表現を追加しない。
- 仕様確認なしに設問数、軸、タイプ数を変更しない。
- テスト失敗を隠さない。

## 仕様判断に迷ったとき

1. この文書
2. 最新の `docs/handoffs/`
3. `WORKING_LOCATION.md`
4. `VISUAL_FIX_BRIEF.md` などの個別指示書
5. 既存実装
6. 人間への確認

未確認のことは `未確認` と書いてください。

## AIへの短い固定指示

```text
このプロジェクトでは、診断体験の一貫性と投資助言に見えない安全な表現を最優先してください。
未確認事項を推測せず、仕様変更が必要に見える場合は実装前に確認してください。
便利そうという理由だけで機能を追加しないでください。
```
