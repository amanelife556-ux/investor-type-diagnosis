# QA担当 指示書: テキスト品質と表示確認

## 担当

- QA担当: Raman

## 目的

改善されたテキストが、実際のアプリ画面で読みやすく、誤解なく、崩れなく表示されているか確認する。

## 対象

- アプリ全画面
- イントロ
- 質問画面
- 結果画面
- 共有テキスト
- PC表示
- スマホ表示

## 主な確認観点

- 誤字脱字がないか
- 表記ゆれがないか
- 長すぎる文章で画面が読みにくくなっていないか
- ボタン、カード、タグ内で文字がはみ出していないか
- スマホ幅でテキストが重なっていないか
- 投資助言や断定に見えすぎる表現がないか
- 高リスクタイプを煽っていないか
- 低リスクタイプを下げる表現になっていないか
- `preferredMethods` が初心者にも意味を推測できるか
- 共有テキストが自然に読めるか

## 手動確認の推奨パターン

最低限、以下のタイプを確認する。

- SLPG: 低リスク/長期/自動化/値上がり
- SLAI: 低リスク/長期/能動運用/インカム
- RLAG: 高リスク/長期/能動運用/値上がり
- RTPG: 高リスク/短期/自動化/値上がり
- RTAG: 高リスク/短期/能動運用/値上がり

可能なら16タイプすべて確認する。

## 実行する確認

```bash
npm test
node src/__tests__/diagnosisLogic.node-test.mjs
npm run build
npm run dev
```

ブラウザで以下を確認する。

- PC幅
- スマホ幅
- 診断開始から結果表示まで
- もう一度診断する
- X共有テキスト生成
- コピー
- X共有リンク

## 変更してよいこと

- 明らかな誤字脱字の修正
- 表記ゆれの小修正
- QAメモの作成

## 変更しないこと

- 診断ロジックを変更しない
- 設問の意味を変えない
- 文章を大幅に書き換えない
- デザイン方針を変えない

## 成果物

以下を提出する。

- QA結果
- 確認した画面幅
- 確認したタイプコード
- 見つけた問題
- 修正した場合は変更ファイル一覧
- 未確認事項
- 実行したテストと結果

## 完了条件

- 主要タイプの結果画面がPC/スマホで読める
- 共有導線が壊れていない
- テキスト起因の重大な誤解や表示崩れがない
