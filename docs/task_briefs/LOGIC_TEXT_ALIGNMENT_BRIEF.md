# ロジック担当 指示書: テキストと診断軸の整合性確認

## 担当

- ロジック担当

## 目的

コンテンツ担当が改善した文章が、診断ロジック、4軸、16タイプコードの意味とズレていないか確認する。

文章の美しさではなく、「診断結果として正しく読めるか」を見る。

## 対象ファイル

- `src/diagnosisData.ts`
- `src/diagnosisLogic.ts`
- `src/types.ts`
- `src/__tests__/diagnosisLogic.test.ts`
- `src/__tests__/diagnosisLogic.node-test.mjs`

## 確認する軸

現在の4軸は以下。

- risk: 安定重視 `S` / 成長重視 `R`
- horizon: 短期 `T` / 長期 `L`
- involvement: 自動化 `P` / 能動運用 `A`
- reward: インカム `I` / 値上がり `G`

## 主な確認観点

- 各タイプの文章がコードの意味と合っているか
- 設問文と選択肢が、意図した軸へ自然につながっているか
- `preferredMethods` がタイプコードと矛盾していないか
- `summary`、`strength`、`watchout` が同じ軸解釈で書かれているか
- 相性・補完タイプの説明が、現在の `buildRelatedTypes` と矛盾していないか
- 共有文に表示される `shareLine` が結果タイプとズレていないか

## 変更してよいこと

- テスト追加
- テスト期待値の更新
- 明らかな軸ズレの指摘
- 軸ズレを直すための小さな文言修正

## 変更しないこと

- 人間の確認なしに診断軸を追加・削除しない
- 人間の確認なしにタイプコードの意味を変えない
- 人間の確認なしに設問数を変えない
- コンテンツ担当の文体を大きく書き換えない
- デザイン調整をしない

## 必須確認

以下を実行する。

```bash
npm test
node src/__tests__/diagnosisLogic.node-test.mjs
npm run build
```

## 成果物

以下を提出する。

- 整合性レビュー結果
- 問題があるタイプコードと該当文言
- 修正した場合は変更ファイル一覧
- 未解決の判断事項
- 実行したテストと結果

## 完了条件

- 16タイプの文章が4軸の意味と矛盾していない
- テストが通っている
- 判断が必要な軸変更案がある場合は実装せず、保留事項として報告している
